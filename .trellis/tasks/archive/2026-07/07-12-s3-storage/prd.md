# 磁盘与 S3/MinIO 可配置存储及手动保存配置

## Goal

支持用户在页面上配置并选择“本地磁盘”或“S3 兼容存储”作为文件存储模式，所有文件按自身存储类型正确写入和读取；同时将配置中心改为显式保存，只有点击保存按钮后才持久化配置。

## Confirmed Facts

- 当前文件元数据保存在 PostgreSQL `files` 表，实际文件只写入 `UPLOAD_DIR`。
- 前端只持有稳定的 `storageKey`，文件访问统一先请求 `/api/files/:storageKey/access-url`。
- 当前本地文件内容由 `/api/files/:storageKey/content` 鉴权后流式返回。
- 当前 AI 渠道、模型和生成偏好在字段变化后 700ms 自动保存到服务端。
- 当前 Codex Local URL 与 token 在字段变化时立即写入浏览器 `localStorage`。
- 当前“完成”按钮只关闭弹窗，没有承担保存动作；独立配置页面没有保存按钮。

## Requirements

### Storage modes

- 系统支持两种平级存储模式：`local`（本地磁盘）和 `s3`（S3 兼容协议，包含 AWS S3 与 MinIO）。
- 用户在配置页面选择当前默认存储模式；不存在“旧磁盘模式”概念。
- `files` 表记录每个文件实际使用的存储类型；切换默认模式只影响之后写入的文件，不改变已有文件记录。
- 每个文件绑定具体的存储后端记录；修改 S3 endpoint、bucket 等存储身份配置时创建新的后端记录，旧后端配置继续用于读取已有文件。
- 已有文件继续按各自记录的存储类型读取，因此磁盘文件与 S3 文件可以长期并存。
- 当前默认 S3 写入失败时直接返回具体错误，不自动回退到本地磁盘；切换存储模式必须由管理员显式保存。
- `path` 保持相对路径 / object key 语义，`storageKey` 保持稳定，前端业务数据不感知底层存储实现。

### Unified access

- 前端始终调用 `/api/files/:storageKey/access-url` 获取访问链接，不根据存储类型分支。
- 本地磁盘文件返回带短期 token 的 `/api/files/:storageKey/content` 地址。
- S3 文件返回 S3/MinIO 预签名 GET URL。
- `/api/files/:storageKey/content` 只读取本地磁盘文件。
- 生成任务保存结果、读取参考素材和浏览器上传都通过统一存储能力处理，不能直接假设文件位于本地路径。
- 生成任务创建时绑定当时的默认存储后端，任务完成后写入该后端，不受中途切换默认存储影响。
- 浏览器上传继续由 Fastify 中转，服务端完成类型、大小、哈希和权限校验后写入选定后端；本次不做浏览器预签名直传。

### Storage configuration UI

- 配置中心新增“存储”Tab。
- 本地磁盘路径只读展示当前 `UPLOAD_DIR`，不能在页面修改；路径继续由环境变量和 Docker volume 管理。
- 可配置存储模式、S3 endpoint、浏览器可访问 endpoint、region、bucket、access key、secret key、对象前缀与 path-style 选项。
- 每个 S3 后端必须配置便于识别的名称；列表同时展示 endpoint、bucket、对象前缀和关联文件数量。
- 提供独立的 S3/MinIO“调试连接”按钮，使用页面当前草稿在服务端测试 endpoint、bucket 和写入权限，不需要先保存。
- 调试连接不自动创建 bucket；bucket 不存在或无权限时返回明确错误。测试流程写入、读取并删除临时对象。
- 调试连接只验证服务端到 S3/MinIO 的链路，不验证浏览器公开 endpoint 或 CORS。
- 调试成功后返回专用临时对象的浏览器预签名 URL，由管理员自行打开验证公开域名、TLS、签名和 CORS；服务端不请求公开地址。
- 调试对象不写入正式文件表，并在约 10 分钟后由服务端尽力删除。
- 全局保存不自动执行连接测试，也不要求测试成功后才能保存。
- Secret Key 与当前 AI API Key 使用一致策略：数据库明文保存，读取接口只返回脱敏值和是否已配置，不向浏览器返回明文。
- AI API Key 和 S3 Secret 输入框留空保存时保留原值，输入新值时替换；本任务不提供清除密钥操作。
- 存储配置属于服务端全局配置，仅管理员可查看和修改。
- 整个部署实例共享一套存储模式、bucket 和 S3/MinIO 凭据；对象路径继续使用用户 ID 前缀隔离。
- 存储 Tab 提供折叠的历史 S3 后端列表，显示 endpoint、bucket、对象前缀、是否默认和关联文件数量。
- 历史后端允许更新凭据、公开访问 endpoint 和调试连接；仍有关联文件时不能删除。
- S3 后端使用显式“新增”和“编辑”操作：新增创建新的后端 ID，编辑更新原后端记录，切换默认后端不创建记录。
- 新增并保存 S3 后端不会自动成为默认；管理员必须在草稿中显式选择默认后端并点击全局保存后才切换。
- 已有关联文件的后端锁定 bucket 和对象前缀；凭据、endpoint、公开 endpoint、region 和 path-style 可更新。无关联文件的后端可完整编辑或删除。

### Explicit save

- 配置字段变化只更新页面草稿，不自动写入服务端或浏览器持久化存储。
- 配置中心提供明确的“保存”按钮，点击后才持久化配置。
- 配置页面和配置弹窗使用一个固定的全局“保存”按钮，统一保存所有可见 Tab 的草稿，不提供各 Tab 独立保存。
- Codex Local URL、Connect token 和“执行画布操作前确认”也纳入手动保存；连接、断开仍是即时动作，不代表保存配置。
- 当前隐藏的 WebDAV 配置不纳入本次手动保存改造范围。
- 保存成功和失败必须有明确反馈；失败时保留页面草稿供用户修正重试。
- 关闭弹窗或离开配置页面时，不得把未保存草稿静默写入持久化存储。
- 存在未保存修改时，关闭弹窗或离开配置页面必须确认；选择放弃后恢复最近一次已保存配置，选择继续编辑则保留草稿。
- 生图和视频工作台中的模型、尺寸、质量、时长等参数只作用于当前工作台任务，不改写全局默认配置，也不增加独立保存按钮。

## Acceptance Criteria

- [ ] 管理员可在页面选择本地磁盘或 S3/MinIO，并保存服务端全局存储配置。
- [ ] 非管理员不能读取或修改实例级存储配置。
- [ ] 本地磁盘模式下，新上传和新生成文件写入 `UPLOAD_DIR`，访问链接仍指向本地内容接口。
- [ ] 存储页面可查看当前本地磁盘目录，但不能修改。
- [ ] S3 模式下，新上传和新生成文件写入配置的 bucket，访问链接为可用的短期预签名 URL。
- [ ] 当前 S3 后端不可用时上传和生成结果明确失败，且不会在本地磁盘产生回退文件。
- [ ] 切换默认模式后，切换前的文件仍按文件记录的存储类型正常访问。
- [ ] 修改 S3 endpoint 或 bucket 后，旧 S3 后端中的文件仍可通过原后端配置访问。
- [ ] 管理员可查看历史 S3 后端及关联文件数量，并修复失效凭据；有关联文件的后端不能删除。
- [ ] 新增、编辑和切换默认后端行为明确；有关联文件时不能修改 bucket/对象前缀。
- [ ] 新增 S3 后端不会自动接管新文件写入，只有显式保存默认后端选择后才生效。
- [ ] 生成任务可读取本地或 S3 中的参考素材。
- [ ] 长时间生成任务的结果写入任务创建时绑定的存储后端。
- [ ] 浏览器上传在本地与 S3 模式下复用现有服务端上传入口和校验规则。
- [ ] S3 Secret Key 不通过读取接口明文返回。
- [ ] “调试连接”可以使用未保存的 S3 草稿测试连接和写入权限，并显示具体成功或失败原因。
- [ ] 调试连接不会自动创建 bucket，并会清理成功写入的临时测试对象。
- [ ] 调试结果提供可直接打开的临时对象预签名 URL，服务端不主动请求公开 endpoint。
- [ ] 保存 S3 配置时不自动测试连接。
- [ ] AI 渠道、模型、生成偏好和存储配置不再自动保存，只有点击“保存”后才持久化。
- [ ] Codex Local URL、Connect token 和操作确认开关只有点击“保存”后才写入浏览器持久化存储。
- [ ] 保存失败时页面显示真实原因，未保存内容不丢失。
- [ ] 切换 Tab 不丢失草稿，全局保存成功后所有可见配置一起进入已保存状态。
- [ ] 有未保存修改时关闭弹窗或离开页面会提示确认，放弃修改后恢复已保存状态。
- [ ] 生图/视频工作台的临时参数不会自动或隐式覆盖配置中心保存的默认偏好。

## Out of Scope

- 本任务不要求把磁盘文件批量迁移到 S3。
- 本任务不要求删除本地磁盘存储能力。
- 文件删除继续只设置逻辑删除状态，不物理删除磁盘或 S3 对象；引用检查与垃圾回收另行规划。
- 本任务不改变前端画布、素材和生成记录中的 `storageKey` 数据结构。
- 当前隐藏的 WebDAV 配置不纳入本次范围。
- 浏览器预签名直传和客户端分片上传不纳入本次范围。
