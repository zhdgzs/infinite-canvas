# 快速开始

如果你只是想先把项目完整跑起来，优先使用 Docker Compose。当前完整体验需要 Fastify 后端和 PostgreSQL。

## Vercel 部署

Vercel 只适合静态前端预览；完整的登录、文件、画布、素材和 AI 生成任务需要同时部署 `server` 和 PostgreSQL。

## Docker 启动

```bash
git clone git@github.com:zhdgzs/infinite-canvas.git
cd infinite-canvas
docker compose up -d
```

更新时运行：

```bash
docker compose pull && docker compose up -d
```

启动后访问：

```text
http://localhost:3000
```

首次打开后按页面提示注册第一个 admin 账号。

## 本地构建 Docker 镜像

如果你需要基于当前源码构建镜像：

```bash
docker compose -f docker-compose.local.yml up -d --build
```

## 首次使用建议

- 先注册第一个 admin 账号。
- 进入配置页添加模型渠道，填入自己的 `Base URL`、`API Key` 和模型名。
- 如果需要提示词仓库内容，打开 `/prompts` 会由浏览器前端直连拉取并缓存到 IndexedDB。

## 说明

- 画布、素材、生成记录和 AI 配置以服务端 PostgreSQL 为准。
- 上传和生成的媒体文件保存在 Docker 挂载目录 `./data/uploads`。
- 旧浏览器本地数据不迁移，按新系统重新使用。
